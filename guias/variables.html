¿Qué es una variable?

Una variable es un nombre asociado a un valor guardado en memoria. En lenguajes como C o Java suele explicarse como una "caja" que contiene un valor; en Python es más preciso pensar en ella como una etiqueta pegada a un objeto que ya existe en memoria.

Python
nombre = "Roy"
edad = 30
print(nombre)
print(edad)
Caja vs. etiqueta

Esta diferencia parece sutil, pero es clave: más adelante en esta guía verás que dos "etiquetas" pueden apuntar al mismo objeto, y modificar uno afecta lo que ves a través del otro. Con la metáfora de la "caja" ese comportamiento resulta confuso.

Crear y asignar variables

Para crear una variable en Python basta con escribir su nombre, el operador = y el valor que quieres guardar. No es necesario declarar el tipo por adelantado, a diferencia de otros lenguajes.

Python
ciudad = "Cochabamba"
temperatura = 18.5
es_feriado = False
print(ciudad, temperatura, es_feriado)
Comparación con otros lenguajes

En Java o C escribirías algo como int edad = 20;, indicando el tipo antes del nombre. En Python solo escribes edad = 20: Python infiere el tipo a partir del valor que le asignas.

Convenciones de estilo

Más allá de lo que Python exige para que el código funcione, la comunidad sigue una guía de estilo llamada PEP 8. Para variables, la convención es snake_case: minúsculas separadas por guiones bajos.

EstiloEjemploUso en Python
snake_casenombre_completoVariables y funciones (recomendado)
camelCasenombreCompletoComún en Java/JavaScript, no en Python
PascalCaseNombreCompletoReservado para nombres de clases
Python
nombre_completo = "Ana López"
precio_total = 45.90
print(nombre_completo, precio_total)
Nombres descriptivos

Además del estilo, elige nombres que expliquen qué contienen: promedio_estudiante es mucho más claro que p o x1, sobre todo cuando el programa crece.

Reasignar variables

Una variable puede reasignarse tantas veces como necesites, incluso cambiando de tipo, porque Python es de tipado dinámico.

Python
puntaje = 0
print(puntaje)
puntaje = 10
print(puntaje)
puntaje = puntaje + 5
print(puntaje)
Operadores de asignación aumentada

Escribir puntaje = puntaje + 5 es tan común que Python ofrece una forma corta: puntaje += 5. Ambas líneas hacen exactamente lo mismo.

Python
puntaje = 10
puntaje += 5
print(puntaje)
puntaje -= 2
print(puntaje)
OperadorEquivale a
x += nx = x + n
x -= nx = x - n
x *= nx = x * n
x /= nx = x / n

Asignación múltiple

Python permite asignar varias variables en una sola línea, separando nombres y valores por comas.

Python
nombre, edad, activo = "Roy", 30, True
print(nombre, edad, activo)

También puedes asignar el mismo valor a varias variables a la vez:

Python
x = y = z = 0
print(x, y, z)
Intercambiar valores sin variable temporal

En muchos lenguajes, intercambiar el contenido de dos variables requiere una tercera variable auxiliar. En Python, la asignación múltiple lo resuelve en una sola línea.

Python
a = 5
b = 10
a, b = b, a
print(a, b)

Variables como referencias a objetos

Volvamos a la idea de la "etiqueta": cuando asignas una variable a otra, ambas quedan apuntando al mismo objeto en memoria, no a una copia independiente.

Python
a = [1, 2, 3]
b = a
b.append(4)
print(a)
print(b)
¿Por qué cambió "a" si solo modifiqué "b"?

Porque a y b son dos etiquetas para el mismo objeto lista. Esto no ocurre con tipos inmutables como int, float, str o bool, que nunca cambian de contenido una vez creados.

La función id() te muestra la dirección de memoria de un objeto, y el operador is compara si dos variables apuntan al mismo objeto (a diferencia de ==, que compara si los valores son iguales).

Python
x = [1, 2]
y = [1, 2]
print(x == y)
print(x is y)
Igualdad no es identidad

x e y tienen el mismo contenido (== es True), pero son dos objetos distintos en memoria (is es False). Esta distinción será clave más adelante, cuando trabajes con objetos en POO.

Constantes por convención

Python no tiene una palabra clave como const para crear constantes verdaderas. En su lugar, la comunidad usa una convención: los nombres en MAYÚSCULAS indican que ese valor no debería cambiar.

Python
PI = 3.14159
IVA = 0.13
MAX_INTENTOS = 3
print(PI, IVA, MAX_INTENTOS)
Solo una convención

Python no impide reasignar PI; nada se rompe técnicamente si lo haces. Es una señal para otros programadores (y para ti mismo) de que ese valor está pensado para no cambiar durante la ejecución del programa.

Errores comunes

❌ Usar una variable antes de crearla

print(saldo) antes de que saldo exista produce NameError: Python lee el programa de arriba hacia abajo.

❌ Empezar el nombre con un número

2nombre = "x" produce SyntaxError. Los nombres deben empezar con letra o guion bajo.

❌ Usar una palabra reservada

class = "A" produce SyntaxError: class, for, if, etc. están reservadas por el lenguaje.

¿Sabías que...?




🔢 Python cachea enteros pequeños
CPython guarda en caché los enteros entre -5 y 256: dos variables con ese mismo valor pequeño pueden compartir el mismo objeto en memoria, y por eso x is y a veces da True sin que lo esperes.


🐫 camelCase no es "pythónico"
La guía de estilo oficial PEP 8 recomienda snake_case para variables; camelCase es mucho más común en Java y JavaScript.


🔒 Python no tiene constantes reales
A diferencia de lenguajes con const o final, en Python cualquier variable —incluidas las "constantes" en mayúsculas— puede reasignarse. El respeto a esa convención es solo cultura de equipo.
